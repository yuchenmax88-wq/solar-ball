## 2. 硬件引脚连接

### 总引脚图

```
                    ESP32-WROOM-32 (38-pin dev board)
                   ┌────────────────────────────┐
       BOOT ───────┤ GPIO 0              GPIO 23├────── SIM7600G PWR_KEY
     MIP_MOSI ──────┤ GPIO 2              GPIO 22├────── I2C1 SCL (ADS1115#1)
     MIP_SCLK ──────┤ GPIO 4              GPIO 21├────── I2C1 SDA (ADS1115#1)
      MIP_CS ───────┤ GPIO 5              GPIO 19├────── I2C2 SCL (ADS1115#2)
    MUX_EN2 ───────┤ GPIO 12             GPIO 18├────── I2C2 SDA (ADS1115#2)
    MUX_EN3 ───────┤ GPIO 13             GPIO 17├────── SIM7600G TX
    MUX_EN1 ───────┤ GPIO 14             GPIO 16├────── SIM7600G RX
    MUX_EN4 ───────┤ GPIO 15             GPIO 36├────── BAT_ADC (电池电压)
     3.3V ─────────┤ 3V3                 GPIO 32├────── MUX_S0
       GND ────────┤ GND                 GPIO 33├────── MUX_S1
                   │                     GPIO 25├────── MUX_S2
                   │                     GPIO 26├────── MUX_S3
                   │                     GPIO 27├────── MUX_EN0
                   │                     GPIO 14├────── MUX_EN1
                   │                     EN (RST)├── 复位
                   │                     5V/VIN  ├── 5V 供电输入
                   └────────────────────────────┘
```

### 引脚速查表

#### MUX 控制线（5 个 MUX 共用 S0–S3）

| ESP32 GPIO | 连接目标 | 说明 |
|:----------:|---------|------|
| **32** | 全部 MUX 的 S0 脚 | 通道选择 bit 0 |
| **33** | 全部 MUX 的 S1 脚 | 通道选择 bit 1 |
| **25** | 全部 MUX 的 S2 脚 | 通道选择 bit 2 |
| **26** | 全部 MUX 的 S3 脚 | 通道选择 bit 3 |

#### MUX 使能线（每个 MUX 单独一个，低电平有效）

| ESP32 GPIO | 连接目标 | 负责的传感器 |
|:----------:|---------|:----------:|
| **27** | MUX #1 EN 脚 | Sensor 0–15 |
| **14** | MUX #2 EN 脚 | Sensor 16–31 |
| **12** | MUX #3 EN 脚 | Sensor 32–47 |
| **13** | MUX #4 EN 脚 | Sensor 48–63 |
| **15** | MUX #5 EN 脚 | Sensor 64–79 |

#### I2C 总线

| ESP32 GPIO | 连接目标 | I2C 地址 |
|:----------:|---------|:--------:|
| **21** (SDA) | ADS1115 #1 SDA | 0x48 |
| **22** (SCL) | ADS1115 #1 SCL | — |
| **18** (SDA) | ADS1115 #2 SDA | 0x49 |
| **19** (SCL) | ADS1115 #2 SCL | — |

> **注意**：I2C 需要 4.7kΩ 上拉电阻到 3.3V（SDA 和 SCL 各一个）。

#### 4G 模块（SIM7600G）

| ESP32 GPIO | SIM7600G 引脚 | 说明 |
|:----------:|:------------:|------|
| **17** | TX（模块 RX） | ESP32 → 模块 |
| **16** | RX（模块 TX） | 模块 → ESP32 |
| **23** | PWR_KEY | 上电脉冲（拉低 1.5 秒） |
| **2** | RESET | 硬件复位（目前代码未用） |

#### 其他

| ESP32 GPIO | 连接目标 | 说明 |
|:----------:|---------|------|
| **36** | 电池分压中点 | 两个 100kΩ 串联，中点接 GPIO36 |
| **0** | 按键（BOOT） | 低电平 = 按下 = 进入校准模式 |
| **2** | MIP MOSI | Sharp Memory LCD 数据线 (SPI) |
| **4** | MIP SCLK | Sharp Memory LCD 时钟线 (SPI) |
| **5** | MIP CS | Sharp Memory LCD 片选线 (SPI) |

## 3. 电路接线速查

### 单个传感器接线（重复 80 次）

```
     3.3V ─── 10kΩ ─┬── MUX 输入脚（例如 MUX#1 的 Y0）
                     │
                  ┌──┴──┐
                  │ ALS │  ALS-PT19 光电晶体管
                  │PT19 │  （长脚 = 集电极接电阻，短脚 = 发射极接地）
                  └──┬──┘
                     │
                    GND
```

> **关键**：长脚（集电极）接 10kΩ→3.3V 和 MUX 输入；短脚（发射极）接地。  
> **绝对不要反接**，否则烧毁传感器。

### MUX → ADC 接线

```
MUX #1 公共脚 ─── ADS1115#1 (0x48) AIN0
MUX #2 公共脚 ─── ADS1115#1 (0x48) AIN1
MUX #3 公共脚 ─── ADS1115#2 (0x49) AIN0
MUX #4 公共脚 ─── ADS1115#2 (0x49) AIN1
MUX #5 公共脚 ─── ADS1115#2 (0x49) AIN2
```

### 电源接线（按顺序接，接一步测一步）

```
太阳能 5V ──→ TP4056 IN+  ──→ TP4056 BAT+ ──→ 18650 正极
                                     BAT- ──→ 18650 负极
              TP4056 OUT+ ──→ MT3608 IN+  ──→ MT3608 OUT+ ─┬──→ AMS1117 IN  ──→ 3.3V（给 ESP32 + 传感器）
                          MT3608 IN-  ──→ MT3608 OUT- ─┼──→ SIM7600G 5V 输入
                                                         └──→ 全部 GND 共地

⚠ 接电池前用万用表确认正负极，反接会烧毁所有电路。
⚠ MT3608 先调到 5.0V 再接负载（旋转电位器，万用表测量）。
```

## 4. 组装步骤概览

> 详细步骤见 `docs/assembly_guide.md`

1. **焊 PCB**：先焊电阻（10kΩ×80 + 100kΩ×2 + 4.7kΩ×2），再焊 IC 座和排针
2. **焊传感器**：每个 ALS-PT19 焊 10cm 杜邦线 + 10kΩ 电阻，套热缩管
3. **装传感器到球壳**：从球壳内侧插入传感器孔，热熔胶固定，镜头与外壳齐平
4. **装内部框架**：固定 ESP32、MUX 板、ADS1115、SIM7600G、电池、电源模块
5. **接 4G 天线**：SMA 转 IPEX 线连接 SIM7600G，天线磁吸底座固定在外壳上
6. **插 SIM 卡**：SIM7600G 的 SIM 卡槽
7. **通电测试**：用 USB 供电，检查串口输出是否正常启动
8. **密封**：硅胶涂在所有传感器孔和球壳接缝处，晾干 24 小时
9. **安装太阳能板**：贴在球顶平坦区域，引线接 TP4056 输入
10. **装杆**：球底部开口套在支撑杆上，确保球顶朝上

## 5. 安装电脑软件

打开终端（Windows 按 `Win+R`，输入 `cmd`，回车）：

```bash
# 安装 PlatformIO（用于编译和烧录固件）
pip install platformio

# 安装项目依赖（校准工具 + 接收端）
cd D:\my-project\solar-ball\tools
pip install -r requirements.txt
```

## 6. 烧录固件到 ESP32

1. 用 Micro-USB 线连接 ESP32 和电脑
2. 确认串口号（Windows 在"设备管理器 → 端口"中查看，通常是 **COM3**）

编辑 `firmware/platformio.ini`，把 `upload_port` 改成你的串口号：

```ini
upload_port = COM3      # 改成你的
monitor_port = COM3     # 改成你的
```

顺便检查 `firmware/main/config.h` 中的球 ID 和 MQTT 服务器：

```c
#define BALL_ID             "ball-001"     // 改你的球编号
#define MQTT_BROKER_HOST    "broker.emqx.io"  // 公共测试服务器，或自建
```

3. 编译 + 烧录：

```bash
cd D:\my-project\solar-ball\firmware
platformio run --target upload
```

看到 `Success` 即烧录完成。

4. 验证固件运行（可选）：

```bash
platformio device monitor
```

应该看到：

```
Solar Ball v1.1.0 booting...
Ball ID: ball-001
No calibration data found. Using default normalization.
```

## 7. 户外自动校准

> **前提**：球已组装好、SIM 卡已插入、在户外（能看见天空）。

### 方法一：一键自动（推荐）

把球放在户外，USB 线连电脑：

```bash
cd D:\my-project\solar-ball\tools
python auto_calibrate.py COM3 --auto
```

球会自动：
1. 连接 4G 网络
2. 基站定位（知道自己在哪）
3. NTP 对时（知道现在几点）
4. 计算太阳精确位置
5. 扫描 80 个传感器
6. 自动建立映射
7. 保存到永久存储

约 60 秒完成。校准后拔掉 USB，球开始自主工作。

### 方法二：手动（阴天备用）

1. 按住 BOOT 键（GPIO 0 的按钮）同时上电，进入校准模式
2. 打开任意串口工具（波特率 115200），发送 `CAL:SUN_AUTO`
3. 或者走 80 步手电筒流程：发送 `CAL:START`，按提示逐个位置照手电筒

也可以运行 PC 工具引导：

```bash
python auto_calibrate.py COM3          # 手电筒模式（不带 --auto）
```

### 可选：基线校准

在校准模式下发送：

```
CAL:BASELINE_START
```

把球放在均匀散射光下（阴天户外或白布覆盖），自动采样 100 次建立基线。

## 8. 启动接收端看数据

球正常工作后（每 5 秒发一次数据），在电脑上运行：

```bash
cd D:\my-project\solar-ball\tools
python mqtt_receiver.py
```

你会看到：

```
[ball-001] 14:32:05 Sun: az= 169.2° elev=+73.0°  Panel: az= 169.2° tilt=17.0°  SOC=87% RSSI=-89dBm conf=220
  Faults: none  Flags: [DIR_OK CALIB ]  vector:OK

[ball-001] 14:32:10 Sun: az= 169.5° elev=+72.8°  Panel: az= 169.5° tilt=17.2°  SOC=87% RSSI=-89dBm conf=220
  Faults: none  Flags: [DIR_OK CALIB ]  vector:OK
```

| 输出项 | 含义 |
|--------|------|
| `az=169° elev=+73°` | 太阳在方位角 169°（正南偏东），仰角 73° |
| `Panel: az=169° tilt=17°` | **直接发给电机控制器的值**：水平转 169°，倾斜 17° |
| `SOC=87%` | 电池剩 87% |
| `RSSI=-89dBm` | 4G 信号强度，-90 以上为良好 |
| `conf=220` | 方向可信度 220/255（>200 很可靠，<50 不要用） |
| `Faults: none` | 无故障，一切正常 |
| `DIR_OK CALIB` | 方向有效 + 已校准 |

> **如果出现 Faults**：对照 README 中的故障码表排查。`LOW_BAT` 说明要充电，`SENSOR_OPEN` 说明某个传感器松动或脏了。

## 9. 连接太阳能阵列电机

接收端输出的 `panel_az` 和 `panel_tilt` 就是电机需要转到的角度：

| 参数 | 物理含义 | 电机动作 |
|------|---------|---------|
| `panel_az` | 水平旋转角度（0°=北, 90°=东, 180°=南） | **方位电机**转到这个角度 |
| `panel_tilt` | 倾斜角度（0°=竖直, 90°=水平朝上） | **俯仰电机**转到这个角度 |

### 如何连（通用方案）

`mqtt_receiver.py` 第 128 行打印完数据后，加上你的电机控制代码即可。例如：

- **串口步进电机控制器**：`serial.write(f"AZ={panel_az}\nTILT={panel_tilt}\n")`
- **Modbus RTU**：写入保持寄存器 0x0000=panel_az, 0x0001=panel_tilt
- **CAN 总线**：发送标准帧 ID=0x100, data=[az_hi, az_lo, tilt_hi, tilt_lo]

具体协议取决于你用的电机驱动器型号（联系厂家要协议文档）。

## 10. 日常运行与维护

### 正常运行

球完全自治：
- 每 5 秒：唤醒 → 扫描 → 诊断 → 计算 → 显示 → 发送 → 休眠
- 休眠期间功耗 < 1mA
- 有阳光时充电，电池保持在 80–100%
- MIP 反射式液晶屏：刷新后无需电力维持图像。ESP32 休眠期间屏幕持续显示最新状态，功耗为零，阳光下清晰可读

### 你只需要做的事

| 频率 | 事项 |
|------|------|
| 每天 | 看一眼接收端有没有红色 Faults |
| 每月 | 检查球壳表面是否干净 |
| 每半年 | 换 PTFE 防沙套（¥5，见 `hardware/dust_protection.md`） |
| 每年 | 换 IoT SIM 卡续费 |

### 阴天/夜晚

球会自动检测：
- **阴天**：`FLAG_OVERCAST` 置位，方向指天 `(0,0,1)`，可信度降低
- **夜晚**：`FLAG_NIGHT` 置位，方向指天，可信度极低
- 接收端根据 `conf` 决定是否转动阵列（conf<50 可以不动，省电）

## 11. 常见问题排查

| 现象 | 可能原因 | 解决办法 |
|------|---------|---------|
| 串口无输出 | 波特率不对 / USB 线只供电 | 确认 115200 波特率，换数据线 |
| 所有传感器读 0 | 3.3V 供电没接 / I2C 不通 | 万用表量 3.3V 和 I2C 上拉电阻 |
| 某个传感器永远最亮/最暗 | 焊接短路或断路 | 万用表蜂鸣档测该传感器两个脚到 MUX 的线 |
| 4G 初始化失败 | SIM 卡没插好 / 没信号 | 检查 SIM 卡方向，到开阔处重试 |
| 校准后方向不准 | 校准那天是阴天 / 球没放正 | 重新在晴天下校准 |
| MQTT 发不出去 | 没网络 / Broker 连不上 | broker.emqx.io 有时会限流，换自建 Broker |
| 屏幕不显示 | SPI 接线错误 | 检查 GPIO 2/4/5 到 MIP 屏的接线，确认信号通断 |
| 电池很快没电 | 太阳能板脏了 / MT3608 没调好 | 清洁板面，检查 MT3608 输出是否 5V |
